# Acceptance Checklist

Use this checklist before calling an active task complete.

- [ ] The requested behavior or documentation exists.
- [ ] The change stays within the active task scope.
- [ ] Tests were added or updated when behavior changed.
- [ ] Targeted tests pass.
- [ ] The default verifier passes.
- [ ] Any milestone verifier passes.
- [ ] Optional live checks are reported separately.
- [ ] Docs match the implemented behavior.
- [ ] No secrets or generated junk were staged.
- [ ] The final report includes exact verification outcomes.
